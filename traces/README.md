# Trace inventory

Captured latency traces (tap schema, see `src/trace.rs`). The directory is
gitignored except for this file: traces are measurement data, some are large,
and token-recording ones carry model content. This README is the index of
what each capture is and how it's used; keep it current when adding captures.

Capture hygiene rules learned the hard way:

- One workload per pod lifetime; a half-rewired pod records garbage.
- Never fit and gate on the same trace.
- Never trust a single-seed gate: 3 of ~10 multiturn captures turned out
  anomalous (`mtfit`, `nocache3`, `nocachelo`), which only multi-seed
  averaging exposed.
- Fetch promptly; the cluster reaps idle GPU pods (~35 min) and emptyDir
  traces die with the pod.

## local-sim/

Tap self-captures (tap recording the simulator itself, no GPU): `burst`,
`multiturn`, `poisson` arrival patterns. These are the ground-truth runs for
the README's workload-scenario figures (`replay-arrivals-*.png`,
`multiturn-cache-effect.png`), regenerated by `just figures`.

Captured 2026-06-12 on the step-model engine: vllm-rs (protocol pin
c9340e6f3) → tap → vllm-vcr play, latency model fit from the canonical set
(`h200-sweep-full` + `h200-multiturn-mtfit2` + `h200-multiturn-nocache4`),
engine at default scheduler limits (multiturn additionally
`--kv-cache-size 65536`). Seeded loadgen
(seed 0): poisson rate 4 x 120s (464 reqs), burst 24 per 10s x 120s (288),
multiturn rate 1 x 5 turns x two 10k prefixes (495). Replay legs must mirror
the same fit and engine flags (see `scripts/make_figures.sh`); fitting from a
trace without cached/deep-context records breaks the warm-multiturn cohorts.

Captures from the pre-step-model engine were replaced in place; they are not
reproducible on HEAD.
